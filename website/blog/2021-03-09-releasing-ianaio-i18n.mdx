Today, we officially release **[Ianaio 2 i18n](https://ianaio.io/docs/i18n/introduction)** ([internationalization](https://en.wikipedia.org/wiki/Internationalization_and_localization)), as part of [2.0.0-alpha.71](https://github.com/ianaio/ianaio/releases/tag/v2.0.0-alpha.71).

Ianaio 2 has now reached **full feature parity with Ianaio 1**. 🎉 And soon, after a few additional infrastructure updates and a bit more testing, the **first Ianaio 2 beta** will be released.

**We went the extra mile**, and the new i18n support is **even better than in Ianaio 1**.

In this post, we will present you the **translation workflow**, explain some **design decisions** and **showcase early adopter sites** that just went live: [Jest](https://jestjs.io/), [Redwood](https://learn.redwoodjs.com/), and [Datagit](https://datagit.ir/).

We also **dogfood** the i18n support on the **Ianaio 2 site itself**, and this post is already available in [English](https://ianaio.io/blog/2021/03/09/releasing-ianaio-i18n) and [French](https://ianaio.io/fr/blog/2021/03/09/releasing-ianaio-i18n)!

## Translate your site

You can translate a Ianaio site in **3 simple steps**:

1. **Configure**: declare the default locale and alternative locales in `ianaio.config.js`
1. **Translate**: put the translation files at the [correct filesystem location](https://ianaio.io/docs/next/i18n/introduction#translation-files-location)
1. **Deploy**: build and deploy your site using a single or multi-domain strategy

The i18n support is **very flexible** and based on the **filesystem**.

The **[i18n tutorial](https://ianaio.io/docs/i18n/tutorial)** is the best way to get started, and we provide help to use **[Git](https://ianaio.io/docs/i18n/git)** or **[Crowdin](https://ianaio.io/docs/i18n/crowdin)**.

## Design decisions

The goals of the Ianaio i18n system are:

- **Simple**: just put the translated files in the [correct filesystem location](https://ianaio.io/docs/next/i18n/introduction#translation-files-location)
- **Flexible translation workflows**: use Git (monorepo, forks, or submodules), SaaS software, FTP
- **Flexible deployment options**: single, multiple domains, or hybrid
- **Modular**: allow plugin authors to provide i18n support
- **Low-overhead runtime**: documentation is mostly static and does not require a heavy JS library or polyfills
- **Scalable build-times**: allow building and deploying localized sites independently
- **Localize assets**: an image of your site might contain text that should be translated
- **No coupling**: not forced to use any SaaS, yet integrations are possible
- **Easy to use with [Crowdin](https://crowdin.com/)**: multiple Ianaio 1 sites use Crowdin, and should be able to migrate to v2
- **Good SEO defaults**: we set useful SEO headers like [`hreflang`](https://developers.google.com/search/docs/advanced/crawling/localized-versions) for you
- **RTL support**: locales reading right-to-left (Arabic, Hebrew, etc.) are supported and easy to implement
- **Default translations**: classic theme labels are translated for you in [many languages](https://github.com/ianaio/ianaio/tree/main/packages/ianaio-theme-translations/locales)

## Showcase

We worked with a few volunteers to **dogfood the i18n support** before releasing it.

### Jest: upgrading Ianaio

We have made it possible to **upgrade** a **translated Ianaio 1 site** to Ianaio 2.

**[Jest](https://jestjs.io)**, using Ianaio 1 and Crowdin was one of those sites.

The Ianaio 2 migration was successful and the new site is now deployed in production ([announcement post](https://jestjs.io/blog/2021/03/09/jest-website-upgrade)).

[![Jest's website front page in Japanese](/img/blog/2021-03-09-releasing-ianaio-i18n/jest.png)](https://jestjs.io)

### Redwood: adopting Ianaio

[Redwood](https://redwoodjs.com/) is a React full-stack Jamstack framework.

They were looking for a solution to create an **internationalized learning platform**, and became early adopters of Ianaio 2 i18n for [learn.redwoodjs.com](https://learn.redwoodjs.com/), using Crowdin to get it translated to French.

[![Redwood's doc page in French](/img/blog/2021-03-09-releasing-ianaio-i18n/redwood.png)](https://learn.redwoodjs.com/)

### Datagit: using LTR support

The i18n system should work with any language, including **Right-to-Left languages**.

[Datagit.ir](https://datagit.ir/) is using Farsi, and a simple configuration option is able to **flip the Ianaio theme** to add the required Right-to-Left support.

[![Datagit's website in Persian, a right-to-left language. The sidebar appears on the right of the window and the TOC appears on the left.](/img/blog/2021-03-09-releasing-ianaio-i18n/datagit.png)](https://datagit.ir/)
