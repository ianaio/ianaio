# 📦 eslint-plugin

[ESLint](https://eslint.org/) is a tool that statically analyzes your code and reports problems or suggests best practices through editor hints and command line. Ianaio provides an ESLint plugin to enforce best Ianaio practices.

## Installation

```bash npm2yarn
npm install --save-dev @ianaio/eslint-plugin
```

## Usage

Add `@ianaio` to the plugins section of your `.eslintrc` configuration file:

```json title=".eslintrc"
{
  "plugins": ["@ianaio"]
}
```

Then, you can extend one of the configs (e.g. the `recommended` config):

```json title=".eslintrc"
{
  "extends": ["plugin:@ianaio/recommended"]
}
```

Each config contains a set of rules. For more fine-grained control, you can also configure the rules you want to use directly:

```json title=".eslintrc"
{
  "rules": {
    "@ianaio/string-literal-i18n-messages": "error",
    "@ianaio/no-untranslated-text": "warn"
  }
}
```

## Supported Configs

- Recommended: recommended rule set for most Ianaio sites that should be extended from.
- All: **all** rules enabled. This will change between minor versions, so you should not use this if you want to avoid unexpected breaking changes.

## Supported Rules

| Name | Description |  |
| --- | --- | --- |
| [`@ianaio/no-untranslated-text`](./no-untranslated-text.mdx) | Enforce text labels in JSX to be wrapped by translate calls |  |
| [`@ianaio/string-literal-i18n-messages`](./string-literal-i18n-messages.mdx) | Enforce translate APIs to be called on plain text labels | ✅ |

✅ = recommended

## Example configuration

Here's an example configuration:

```js title=".eslintrc.js"
module.exports = {
  extends: ['plugin:@ianaio/recommended'],
  plugins: ['@ianaio'],
  rules: {
    '@ianaio/no-untranslated-text': [
      'warn',
      {ignoredStrings: ['·', '—', '×']},
    ],
  },
};
```
